## Key decisions

- **Synchronous `better-sqlite3`**: all DB calls are synchronous; adapter methods return `Promise.resolve(result)` to satisfy the async `StorageAdapter` interface without introducing actual async patterns
- **In-memory SQLite for tests**: `:memory:` database per test — no file I/O, full isolation, no cleanup needed
- **Field name mapping in adapter layer**: TypeScript `tasted_at` → SQLite `date`, `tags` → `extracted_tags`, `quality_assessment` → `wset_conclusion_quality` — the rest of the app never sees SQLite column names
- **SheetsAdapter retained** in `backend/sheets/` for reference — not in the active code path
- **`date_first_consumed` is set-once** — written only when `tag_consumed` transitions `false → true` on first note save; never overwritten
- **Credentials file remediation**: `config/wine-project-*.json` was committed in an early commit; removed with `git filter-branch --index-filter` and force-pushed
